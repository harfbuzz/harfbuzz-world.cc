Welcome

HarfBuzz World is a live playground for HarfBuzz, plus harfbuzz-world.cc — a single-file build of HarfBuzz that drops into your own C or C++ project without any build-system glue. The shared Text, Size, and Font controls above feed every demo:

  • shape — the shaped glyph stream as JSON, alongside an SVG preview.
  • subset — produce a font subset for the current text and download it.
  • raster — pixel-perfect BGRA rendering at any size, blitted to a canvas.
  • vector — the same shaped text rendered to SVG and downloadable PDF.
  • gpu — slug-based GPU rendering via WebGL2 / WebGPU.

Embedding harfbuzz-world.cc

Add HarfBuzz to your project as a submodule (or vendor a checkout next to your code). Then point your build at one file inside it — src/harfbuzz-world.cc, the amalgamation that #includes every .cc in the library so the whole thing compiles as a single translation unit:

c++ -std=c++11 -O2 \
    -I third_party/harfbuzz/src \
    -c third_party/harfbuzz/src/harfbuzz-world.cc \
    -o harfbuzz-world.o

Link harfbuzz-world.o with the rest of your program and call the public C API as usual:

#include <hb.h>
#include <hb-ot.h>

hb_blob_t  *blob = hb_blob_create_from_file ("MyFont.ttf");
hb_face_t  *face = hb_face_create (blob, 0);
hb_font_t  *font = hb_font_create (face);
hb_buffer_t *buf = hb_buffer_create ();

hb_buffer_add_utf8 (buf, "Hello", -1, 0, -1);
hb_buffer_guess_segment_properties (buf);
hb_shape (font, buf, NULL, 0);

unsigned len;
hb_glyph_info_t      *info = hb_buffer_get_glyph_infos     (buf, &len);
hb_glyph_position_t  *pos  = hb_buffer_get_glyph_positions (buf, NULL);
/* …draw the glyphs… */

Pick what you need at compile time with feature defines: HB_TINY for a minimal core, HB_HAS_RASTER, HB_HAS_VECTOR, HB_HAS_GPU, HB_HAS_SUBSET to opt into the matching subsystems.
